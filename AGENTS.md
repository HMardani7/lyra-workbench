# Lyra Workbench Rules

Lyra may write files in this repository.

Use this repo for:
- Ideas
- Research notes
- Task breakdowns
- Implementation plans
- PR drafts
- Questions to ask Hammad later
- durable agent/project knowledge when it belongs in the Obsidian vault

Do not store secrets, API keys, passwords, tokens, private credentials, or copied `.env` contents.

When discussing code changes for actual app repos, write implementation plans or patch drafts here. Do not directly modify production repositories.

## Context and token discipline

This workbench is a curated knowledge base, not a raw transcript dump.

Before project work, read:
1. `01-Knowledge/Context/Preferences.md`
2. `01-Knowledge/Context/Environment.md`
3. `01-Knowledge/Context/Active Work.md`
4. `01-Knowledge/Context/Vault Operating Rules.md`
5. targeted project overview/index files only

Do not bulk-read all markdown files. Use search first, then read the smallest set of relevant files.

Long research belongs in `04-Research/` with a concise README/index. Stable conclusions should be distilled into `01-Knowledge/` or `02-Projects/`. Superseded material belongs in `08-Archive/`.
